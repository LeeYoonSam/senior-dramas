# 예시 2: 빠른 대본 생성

이미 컨셉이 있는 상태에서 빠르게 대본만 생성하는 예시입니다.

## 시나리오

기존 프로젝트의 특정 챕터 대본만 재생성하거나, 컨셉을 바탕으로 빠르게 대본을 뽑아내는 경우입니다.

---

## Case 1: 기존 프로젝트 대본 재생성

### 상황
- 이미 `story-concept.yaml`과 `chapter-blueprint.yaml`이 있음
- 3장 대본만 다시 생성하고 싶음

### 실행

```
사용자: /drama:script --chapter 3
```

```
미영: "시어머니의마음" 프로젝트의 3장을 재생성할게요.

      기존 블루프린트를 확인했어요:

      3장: 첫 번째 대화
      • 하늘이가 할머니에게 인터뷰 요청
      • 순자, 30년 전 이야기 시작

      생성 중...

      [진행률: ████████████ 100%]

      ━━━━━━━━━━━━━━━━━━━━━━━
      📝 3장: 첫 번째 대화 (재생성)
      ━━━━━━━━━━━━━━━━━━━━━━━

      ## 씬 1: 순자의 아파트 (오후)

      **[내레이션]**
      토요일 오후, 하늘이는 혼자 할머니 집을 찾았다.
      손에는 알록달록한 노트와 펜이 들려 있었다.

      **[대화]**
      (초인종 소리)

      순자: (문 열며) "어머, 하늘이? 웬일이니?"

      하늘: "할머니! 저 인터뷰하러 왔어요!"

      순자: (어리둥절) "인터뷰? 뭔 인터뷰?"

      **[대화]**
      하늘: (노트 펼치며) "학교 숙제예요.
             '우리 가족 이야기'라는 과제인데,
             할머니 이야기도 듣고 싶어서요!"

      순자: (살짝 미소) "이 할미가 무슨 이야기가..."

      하늘: (눈 반짝이며) "할머니는 젊었을 때 어땠어요?
             아빠 어렸을 때요!"

      **[내면 독백 - 순자]**
      '젊었을 때라... 그때가 벌써 30년 전이구나.
       준혁이가 결혼한다고 했을 때...'

      ━━━━━━━━━━━━━━━━━━━━━━━

      파일 저장: scripts/chapter-3.md

      [1] 4장으로 이동
      [2] 3장 수정
      [3] 완료
```

## Case 2: 컨셉만 있고 바로 대본 생성

### 상황
- `story-concept.yaml`만 있음
- 블루프린트 없이 바로 대본 생성하고 싶음

### 실행

```
사용자: /drama:script
```

```
미영: "시어머니의마음" 프로젝트를 확인했어요.

      ⚠️  블루프린트가 없네요.

      [1] 블루프린트 먼저 생성하기 (권장)
      [2] 기본 8장 구조로 바로 대본 생성
```

```
사용자: 2
```

```
미영: 기본 8장 구조로 대본을 생성할게요.

      테마: 화해 (reconciliation)
      기본 구조를 적용합니다...

      [1장 생성 중...]
      [2장 생성 중...]
      ...
      [8장 생성 중...]

      ✓ 전체 대본 생성 완료!

      파일: script-full.md

      주의: 블루프린트 없이 생성된 대본은
      일관성이 다소 떨어질 수 있어요.
      테스트를 실행해보시는 걸 권장해요.

      [1] 테스트 실행
      [2] 대본 확인
      [3] 완료
```

## Case 3: 특정 챕터 범위 생성

### 상황
- 1-4장까지만 먼저 생성하고 싶음

### 실행

```
사용자: /drama:script --chapters 1-4
```

```
미영: 1장부터 4장까지 생성할게요.

      [진행 중...]

      ━━━ 진행 상황 ━━━
      1장: ████████████ 완료
      2장: ████████████ 완료
      3장: ████████░░░░ 80%
      4장: ░░░░░░░░░░░░ 대기

      1장부터 4장까지 완료되었어요!

      생성된 파일:
      • scripts/chapter-1.md
      • scripts/chapter-2.md
      • scripts/chapter-3.md
      • scripts/chapter-4.md

      [1] 5-8장 이어서 생성
      [2] 생성된 대본 확인
      [3] 완료
```

## Case 4: 빠른 단일 챕터 재생성

### 상황
- 테스트에서 5장에 문제가 발견됨
- 5장만 빠르게 재생성

### 실행

```
사용자: /drama:script --chapter 5 --quick
```

```
미영: 5장을 빠르게 재생성합니다.

      블루프린트 기반 빠른 생성 모드...

      [████████████] 완료!

      5장 "진실" 재생성 완료
      파일: scripts/chapter-5.md

      변경 사항:
      • 씬 수: 3개 → 3개 (유지)
      • 대사 수: 45개 → 48개 (증가)
      • 내레이션: 개선됨
```

## 빠른 생성 옵션 정리

| 옵션 | 설명 | 예시 |
|------|------|------|
| `--chapter N` | 특정 챕터만 | `--chapter 3` |
| `--chapters N-M` | 챕터 범위 | `--chapters 1-4` |
| `--quick` | 빠른 모드 | `--chapter 5 --quick` |
| `--project` | 프로젝트 지정 | `--project "시어머니의마음"` |

### 조합 예시

```bash
# 특정 프로젝트의 특정 챕터
/drama:script --project "손녀의선물" --chapter 7

# 빠른 모드로 범위 생성
/drama:script --chapters 5-8 --quick

# 전체 재생성
/drama:script --all
```

## 품질 vs 속도 트레이드오프

| 모드 | 품질 | 속도 | 권장 상황 |
|------|------|------|----------|
| 일반 | ★★★★★ | ★★☆☆☆ | 최종 대본 |
| 빠른 | ★★★☆☆ | ★★★★★ | 초안, 테스트 |
| 챕터별 | ★★★★★ | ★★★☆☆ | 수정, 검토 |

---

## 팁

1. **블루프린트 권장**: 가능하면 블루프린트 먼저 생성
2. **테스트 후 재생성**: 문제 있는 챕터만 재생성
3. **빠른 모드 활용**: 초안 확인 시 빠른 모드 사용
4. **백업**: 재생성 전 기존 파일 자동 백업됨
